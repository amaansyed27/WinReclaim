# Code of Conduct

WinReclaim aims to maintain a technical community where people can report problems, review safety-sensitive code and disagree constructively.

## Expected behaviour

Participants should:

- communicate clearly and respectfully;
- critique code and decisions rather than people;
- provide evidence for technical claims;
- assume good faith while still reviewing safety-critical changes rigorously;
- respect privacy when logs, paths or machine details are involved;
- disclose conflicts of interest;
- accept maintainer decisions about project scope and release safety;
- help newcomers understand the repository's safety constraints.

## Unacceptable behaviour

The following is not acceptable:

- harassment, threats, discrimination or personal attacks;
- publishing another person's private information;
- sharing credentials, private signing keys or sensitive scan data;
- deliberately misleading users about cleanup safety or measured results;
- pressuring maintainers to merge changes that bypass review or tests;
- spam, coordinated disruption or repeated off-topic promotion;
- sexualized language or imagery in project spaces;
- retaliation against good-faith vulnerability reporters.

## Scope

This policy applies to repository issues, pull requests, discussions, release comments and other official project spaces. It also applies when someone publicly represents the project.

## Enforcement

Report conduct concerns privately to the project maintainer through an appropriate private GitHub channel. Do not use the security-reporting channel for ordinary disagreements unless there is also a security issue.

Maintainers may edit or remove content, issue warnings, restrict participation, lock conversations or ban participants. Enforcement decisions consider severity, repetition, impact and willingness to correct the behaviour.

## Technical disagreement

Strong disagreement is allowed. For safety-sensitive topics, contributors are expected to provide reproducible examples, tests, documentation or references. A decision not to merge a change is not misconduct.

## Attribution

This policy is project-specific and may evolve as the contributor community grows.
